# TentaCLAW OS Demo Production Script

## Goal
Produce one clean launch clip that proves the product in under 30 seconds.

## Primary outcome
A viewer should understand:
1. what TentaCLAW OS is
2. that it really works
3. why they should care

## Demo Story
**TentaCLAW OS is HiveOS for AI inference clusters.**

Show that story visually, not by over-explaining it.

## Master Sequence

### Shot 1 — Cold open (2-3s)
**Visual:** Terminal / boot splash / CLAWtopus
**On-screen text:** `HiveOS for AI inference clusters`
**Goal:** product definition immediately

### Shot 2 — Hardware detection (3-4s)
**Visual:** GPU detection lines, node startup, status messages
**On-screen text:** `Auto-detects the hardware`
**Goal:** prove it’s doing real setup work

### Shot 3 — Cluster registration (3-4s)
**Visual:** Farm Hash / node registration / gateway discovery
**On-screen text:** `Joins the cluster`
**Goal:** show this is more than one box and more than a boot splash

### Shot 4 — Dashboard proof (4-5s)
**Visual:** Dashboard with healthy nodes and stats
**On-screen text:** `Health, nodes, and deployment in one place`
**Goal:** visual management proof

### Shot 5 — CLI proof (4-5s)
**Visual:** `clawtopus` command showing status, deploy, or chat
**On-screen text:** `One CLI for the whole tank`
**Goal:** terminal-native credibility

### Shot 6 — Model deploy / inference (5-6s)
**Visual:** deploy command, model available, inference response returns
**On-screen text:** `Deploy. Route. Infer.`
**Goal:** complete the product loop

### Shot 7 — End card (2-3s)
**Visual:** logo / CLAWtopus / dashboard blur / CLI blur
**On-screen text:**
- `TentaCLAW OS`
- `Eight arms. One mind.`
- `Run local.`

## Editing Rules
- Keep cuts tight
- No dead terminal time
- No shaky handheld recording if screen capture is available
- Prefer one clear flow over ten features
- Don’t let the mascot consume more time than the product proof

## Capture Checklist

### Boot / system footage
- [ ] boot splash cleanly visible
- [ ] GPU detection lines readable
- [ ] registration / farm hash visible

### Dashboard footage
- [ ] healthy node cards
- [ ] GPU / VRAM / status visible
- [ ] model or deployment UI visible

### CLI footage
- [ ] `clawtopus status`
- [ ] `clawtopus models` or `clawtopus deploy`
- [ ] one successful inference command if possible

### Optional footage
- [ ] alert state
- [ ] benchmark panel
- [ ] tag filtering
- [ ] model pull progress

## Screenshot Checklist

### Required screenshots
1. **Boot splash**
   - CLAWtopus visible
   - product name visible
   - clean contrast

2. **Dashboard overview**
   - at least 2 nodes
   - healthy cluster vibe
   - enough stats to feel real

3. **CLI control**
   - `clawtopus` command in action
   - tidy terminal theme
   - no clutter

4. **Inference proof**
   - model deployed or response returned
   - obvious that the cluster is doing useful work

### Nice-to-have screenshots
- model search
- pull progress
- alerts
- health score
- tags

## Suggested Screen Text

## Final Test Before Posting
Before publishing the clip, ask:
- Can a stranger tell what this is in under 10 seconds?
- Is there visible proof, not just branding?
- Is the first frame strong enough to stop a scroll?
- Does CLAWtopus support the product instead of distracting from it?

If yes, ship it.
