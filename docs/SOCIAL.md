## Reddit Posts

### r/LocalLLaMA version
**Title:** I built HiveOS for AI inference and it’s actually working

I wanted the HiveOS experience for local AI: boot a node, auto-detect the hardware, register it, deploy models, and manage the whole cluster without hand-holding every machine.

So I built **TentaCLAW OS**.

The stack currently includes:
- gateway + dashboard
- agent daemon
- CLI
- LAN auto-discovery
- model deployment
- tags, alerts, benchmarks
- mock mode so people can test without GPUs

The joke is that it has an octopus mascot named CLAWtopus.
The serious part is that local AI still needs a real operating layer.

Would genuinely love feedback from people running homelabs, Ollama boxes, retired mining rigs, or weird mixed GPU setups.

### r/selfhosted version
**Title:** Flash USB, boot a GPU box, and it joins your AI cluster

I’ve wanted a self-hosted answer to the “how do I manage several local inference nodes without hating my life?” problem.

That turned into **TentaCLAW OS**.

The pitch is simple: **HiveOS for AI inference clusters**.

Flash a node, boot it, let it discover the gateway, register into the cluster, and manage it from a dashboard or CLI.

It’s still early, but the shape is real now and I’d love feedback from people who self-host local AI or have spare hardware they want to put to work.

### Submission text
We built TentaCLAW OS because there’s still a tooling gap between “I own GPUs” and “I can run them as one manageable local inference cluster.”

The project is inspired by HiveOS: boot a node, auto-discover the gateway, register into a farm, deploy models, and manage the cluster from one dashboard or CLI.

Current pieces include a gateway, agent daemon, CLI, dashboard, tagging, alerts, benchmarks, and a mock mode so the full stack can be tested without GPUs.

It’s opinionated, terminal-native, and yes, there’s an octopus mascot named CLAWtopus — but the real goal is practical: make local AI clusters easier to run.

Would love feedback on architecture, launch flow, and what’s still missing for real-world use.
